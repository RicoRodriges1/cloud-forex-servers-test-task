Поточна проблема:

Поки йде запит на сервер, користувач бачить пусті або некоректні ціни.
При помилці запиту немає зрозумілого fallback UI.
Загальний UX виглядає “ламаним” під час loading/error states.

Що потрібно реалізувати:

Loading state
Поки fetch / API request знаходиться в pending state:
НЕ показувати старі або пусті ціни.
Замість цін показувати loader/spinner.
Loader має виглядати акуратно та не ламати layout картки.
Бажано зберегти стабільну висоту елементів, щоб уникнути layout shift.
Кнопки та інтерактивні елементи при потребі зробити disabled.
Error state
Якщо сервер повернув error або request failed:
Замість тарифів показати зрозумілий текст:
“Failed to load pricing”
або “Unable to fetch pricing plans”
Додати кнопку Retry / Try again.
Error state має виглядати частиною дизайну, а не plain text.
Success state
Після успішного response:
плавно відобразити реальні ціни.
прибрати loader без різких стрибків UI.
додати невеликий fade transition якщо доречно.
UX improvements
Не допускати flashing контенту.
Не показувати undefined/null values.
Обробити edge cases:
empty response
partial data
slow network
Код має бути clean та production-ready.
Використати зрозумілу структуру state:
loading
success
error
Tech requirements
React + TypeScript.
Використати conditional rendering.
Якщо вже є state manager або react-query/swr — інтегрувати правильно.
Компонент має бути легко масштабований.
Додатково
